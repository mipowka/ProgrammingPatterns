# Порождающие  (Creational)
Порождающие паттерны фокусируются на процессе создания объектов. 
Эти паттерны помогают создавать объекты в разных ситуациях, скрывая логику создания и делая систему более гибкой. 
Они позволяют создавать объекты, не привязываясь к конкретному типу создаваемого объекта.
### Абстрактная фабрика(Abstract Factory)
Абстрактная фабрика(abstract factory) - паттерн использующийся при проектировании классов таким образом, 
    чтобы облегчить инициализацию и создание новых классов схожих по их функционалу благодаря 
    использованию интерфейсов и абстрактных классов тем самым скрывая логику создания определённого класса
(Помогает создавать объекты определённого типа скрывая логику создания и работая с интерфейсом или
абстрактным классом)
### Строитель(Builder)
Строитель(Builder) - паттерн использующийся для проектирования сложных классов таким образом чтобы
    инициализировать параметры класса можно было не через конструктор потому что он является приватным, 
    а через специальный класс строитель(builder) который будет реализовывать метод build() внутри себя
    и тем самым собирать наш объект с теми параметрами которые нам необходимы
(его основная цель — это обеспечение гибкости в процессе построения объектов с множеством параметров.)
### Прототип(Prototype)
Прототип(Prototype) - паттерн использующийся для проектирования классов таким образом чтобы можно было
    быстро создать копию уже существующего класса. Надо внимательно следить за параметрами клонируемого класса
    чтобы понять применять ли поверхностное клонирование или глубокое
(Паттерн прототип позволяет создать новый объект клонируя уже существующие)
### Одиночка(Singleton)
Одиночка(Singleton) - Паттерн Singleton гарантирует, что класс будет иметь только один экземпляр в системе и
    предоставляет глобальную точку доступа к этому экземпляру.
### Фабричный метод(Factory Method)
Фабрика методом(Factory Method) - паттерн Фабричный метод помогает делегировать создание объектов другим классам, 
    обеспечивая гибкость и расширяемость системы. Этот паттерн широко используется для создания объектов, когда типы 
    объектов могут варьироваться, и важно разделить процесс создания от их использования.
(Паттерн используется для делегирования создания экземпляров другому классу)
# Структурные  (Structural)
Структурные паттерны описывают способы организации классов и объектов в более крупные структуры, 
при этом обеспечивая их удобное взаимодействие. Эти паттерны помогают избежать жесткой связи между классами, 
что улучшает поддержку и расширяемость.
### Адаптер(Adapter)
Адаптер(Adapter) - паттерн Адаптер используется для преобразования интерфейса одного класса в интерфейс, 
    ожидаемый клиентом. Это позволяет объектам с несовместимыми интерфейсами работать вместе, объединяя их 
    функциональность. Таким образом, адаптер позволяет "прикрепить" (или адаптировать) старую систему или 
    интерфейс к новой, не изменяя саму старую систему.
(паттерн адаптер оборачивает нужный нам интерфейс чтобы объединить функционал с другим интерфейсом)
### Прокси(Proxy)
Прокси(Proxy) - Паттерн заменяет какой-то объект чтобы контролировать доступ к другому объекту 
    используется для контроля над объектом(кэширование, ленивая загрузка)
### Мост(Bridge)
Мост(Bridge) - паттерн Мост используется для отделения абстракции и реализации чтобы их можно было изменять
    независимо друг от друга. Добавлять новые функции или классы реализации.
### Компоновщик(Composite)
Компоновщик(Composite) - позволяет комбинировать объекты в иерархии "часть-целое" и работать с ними единообразно. 
    Он позволяет клиенту обращаться с отдельными объектами и их группами одинаково, тем самым упрощая работу 
    с сложными структурами.
(Компонует объекты в древовидные структуры объединяя мелкие компоненты в более крупные)
### Декоратор(Decorator)
Декоратор(Decorator) - паттерн позволяет динамически добавлять новые обязанности (или функциональность) объекту, 
    оборачивая его в другие объекты, которые изменяют или расширяют его поведение.
(динамическое добавление новых обязанностей объекту)
### Фасад(Facade)
Фасад(Facade) - паттерн предоставляет унифицированный интерфейс для работы с несколькими подсистемами, 
    скрывая сложность и взаимодействие между ними
### Приспособленец(Flyweight)
Приспособленец(Flyweight) -  это структурный паттерн проектирования, который используется для оптимизации 
    использования памяти, когда необходимо создать большое количество объектов, которые имеют общие части состояния.
    Вместо того чтобы создавать отдельный объект для каждого элемента, паттерн Flyweight предлагает совместно 
    использовать общие части данных между объектами, а уникальные (особенности) хранятся отдельно.
(оптимизация создания и использования объектов с одинаковыми данными)
# Поведенческие  (Behavioral)
Поведенческие паттерны решают задачи взаимодействия объектов и управления алгоритмами, состоянием и потоками данных.
Эти паттерны помогают наладить коммуникацию между объектами и делают систему более гибкой и расширяемой.
### Цепочка ответственности(Chain of Responsibility)
Цепочка ответственности(Chain of Responsibility) - решает задачу организации обработки запроса через цепочку объектов,
    где каждый объект в цепочке может обработать запрос или передать его дальше по цепочке. Важно, 
    что объект-отправитель не знает, какой конкретный объект получит запрос и будет его обрабатывать, 
    а также не знает, сколько объектов в цепочке будет участвовать в обработке.
(Позволяет передавать запросы вдоль цепочки обработчиков, не связывая отправителя с получателем.)
### Команда(Command)
Команда(Command) - это поведенческий паттерн проектирования, который инкапсулирует запрос как объект, 
    позволяя параметризовать объекты с различными запросами, ставить запросы в очередь, логировать их, 
    а также поддерживать отмену операции. Этот паттерн позволяет отделить объект, который инициирует операцию, 
    от объекта, который знает, как эту операцию выполнить.
(Инкапсулирует запрос как объект, позволяя передавать параметры, очередность выполнения и отмену операций.)
### Интерпретатор(Interpreter)
Интерпретатор(Interpreter) - Паттерн Interpreter определяет представление грамматики языка и использует интерпретатор 
    для обработки и выполнения интерпретаций выражений в этом языке. Важно отметить, что паттерн ориентирован на задачи, 
    где грамматика языка или выражения может быть представлена с помощью рекурсивных структур.
(Определяет грамматику для интерпретации языка и интерпретирует его на основе заданных правил.)
### Итератор(Iterator)
Итератор(Iterator) - предоставляет способ последовательного доступа ко всем элементам составного объекта 
    (например, коллекции), не раскрывая его внутреннюю структуру. Итератор позволяет перебирать элементы объекта, 
    скрывая детали реализации коллекции (например, список, массив, дерево) от клиента.
(получение последовательного доступа ко всем элементам составного объекта скрывая его реализацию)
### Посредник(Mediator)
Посредник(Mediator) - решает задачу инкапсуляции способа взаимодействия множества объектов, предотвращая прямое 
    взаимодействие между ними. Вместо того, чтобы объекты взаимодействовали друг с другом напрямую, все их 
    взаимодействия проходят через посредника. Это позволяет уменьшить сложность системы, поскольку объекты больше 
    не зависят друг от друга напрямую, а взаимодействуют через центральную точку, которая управляет всеми коммуникациями.
### Хранитель(Memento)
Хранитель(Memento) - используется для сохранения и восстановления состояния объекта без раскрытия его внутренней 
    структуры. Этот паттерн полезен, когда нужно сохранить состояние объекта, чтобы позже можно было вернуться 
    к этому состоянию, не нарушая инкапсуляцию.
(фиксирует внутреннее состояние объекта когда нужно сохранить его состояние или его части)
### Наблюдатель(Observer)
Наблюдатель(Observer) - предназначен для определения зависимости один ко многим между объектами, чтобы 
    при изменении состояния одного объекта все зависимые от него объекты были уведомлены и обновились. 
    Этот паттерн является частью группы поведенческих паттернов и широко используется в тех случаях, 
    когда изменение состояния одного объекта должно автоматически приводить к обновлению состояния других объектов.
### Состояние(State)
Состояние(State) - позволяет изменять поведение объекта в зависимости от его внутреннего состояния. 
    Это паттерн поведения, который позволяет объекту изменять свое поведение, когда его внутреннее состояние 
    изменяется, как если бы объект изменял свой класс.
### Стратегия(Strategy)
Стратегия(Strategy)- является паттерном поведения, который позволяет изменять поведение объекта на лету, 
    не изменяя саму структуру объекта. Вместо того чтобы иметь множество условных операторов или 
    сложных конструкций, паттерн позволяет выбрать конкретную стратегию для выполнения определенной задачи
    и делегировать выполнение этой задачи стратегии.
(взаимозаменяемость семейства классов их определение и инкапсулирование каждого из них)
### Шаблонный метод(Template method)
Шаблонный метод(Template method) - это паттерн поведения, который определяет основу алгоритма в методе, 
    оставляя некоторые шаги для реализации в подклассах. Это позволяет подклассам переопределять части 
    алгоритма без изменения его общей структуры.
(определение основы метода и создание возможности подклассам переопределять его части)
### Посетитель(Visitor)
Посетитель(Visitor) - позволяет добавлять новые операции к объектам в структуре без изменения самих объектов.
    Он предоставляет способ выполнения операций с каждым объектом в некоторой структуре данных, не изменяя эти объекты.
(описание действий которые выполняются с каждым объектом в некоторой структуре)
