инкапсуляция способа взаимодействия множества объектов

Один из явных примеров использования паттерна Mediator в Spring — это
    ApplicationEventPublisher и обработка событий через механизм ApplicationEvent.

Пример: ApplicationEventPublisher и ApplicationListener
В Spring события и слушатели событий реализуют паттерн Mediator. Компоненты, генерирующие события, не знают о
    конкретных слушателях, которые должны их обработать. Слушатели, в свою очередь, не знают о том, какой
    компонент инициировал событие. Все взаимодействия происходят через посредника, которым в данном случае
    является ApplicationEventPublisher.

Как это работает:
ApplicationEventPublisher — это интерфейс, который предоставляет метод для публикации событий в контексте Spring.
    Это посредник, через который компоненты отправляют события, не взаимодействуя напрямую друг с другом.

ApplicationListener — это интерфейс, который реализуют компоненты, слушающие события. Эти компоненты
    обрабатывают события, отправленные через посредника.

В Spring можно рассматривать другие механизмы, использующие посредников для упрощения
    взаимодействия между компонентами:

Spring Integration: Использует паттерн Mediator для организации взаимодействия между компонентами
    системы через каналы сообщений.

Spring WebSocket: Взаимодействие между клиентом и сервером через посредника (например, WebSocketMessageBroker),
    который управляет обработкой сообщений.
