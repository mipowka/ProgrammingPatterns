фиксирует внутреннее состояние объекта когда нужно сохранить его состояние или его части

Один из явных примеров использования паттерна Memento в Spring — это поддержка операций Undo/Redo
    или управление состоянием в таких компонентах, как Transaction и Session в различных модулях Spring,
    например, в Spring Data или Spring WebFlow.

Spring WebFlow реализует Memento для сохранения состояний пользовательского интерфейса (UI) в ходе выполнения
    бизнес-процессов или переходов между экранами. В WebFlow вы можете сохранять состояние и восстанавливать его
    при возвращении на предыдущие шаги.

FlowExecution в Spring WebFlow: этот объект хранит информацию о текущем ходе выполнения процесса и
    может быть использован для хранения и восстановления состояния на разных этапах процесса.

Еще один пример использования паттерна Memento в Spring можно найти в механизмах работы с транзакциями в
    Spring Data. В случае работы с базой данных, состояние объектов может быть сохранено и восстановлено на
    разных этапах, особенно если использовать подходы, такие как функции отката (rollback) или контейнеры сессий.

===================================================================================================================
Хотя в Spring нет явного и широко известного класса, который полностью построен на паттерне Memento,
    его элементы можно найти в некоторых компонентах, таких как Spring WebFlow (для сохранения и восстановления
    состояний в процессе выполнения бизнес-процессов) и транзакционные механизмы (где состояние данных может быть
    восстановлено через откат транзакции).

Таким образом, паттерн Memento используется в Spring, например, для сохранения и восстановления
    состояния процесса (как в Spring WebFlow) или для реализации Undo/Redo в более специфичных случаях.
